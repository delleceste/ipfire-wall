Default policy: allow

Each packet inspected by the firewall is examined in search of a permission or denial rule matching its fields (ip addresses, ports, protocol..).
If a match is found, the verdict corresponding to the rule is applied, otherwise the default verdict must be applied.

Here you can decide which default policy is to be applied to the packets that do not match any rule in the firewall: block or accept.

The suggested default policy is to deny all the packets that do not match an explicit rule. This setting represent also a good starting point to safely configure the firewall: at the beginning all the traffic is blocked and little by little you allow the connections you want to.

The popup that notifies the packets not matching any rule, available right clicking on the system tray icon and choosing
Popups...->Unknown Connections
, can help speeding up the selection of the desired and undesired connections through the firewall.

This configuration operates on the /proc entry
/proc/IPFIRE/policy. Read /proc interface for reading and changing default firewall policy from the web documentation of ipfire-wall if you are interested in more details.
