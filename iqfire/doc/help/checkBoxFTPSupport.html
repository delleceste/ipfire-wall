Passive FTP support

To enable the passive FTP support for this rule, you must check this box.

For each packet matching this rule, a new dynamic one is created to allow the passive ftp connection related to the flow permitted by this rule. Usually, this rule concerns the ftp control service, with destination port 21: each ftp control session starts a related ftp data connection, with different port numbers negotiated in the payload data of the control session itself.
