Maximum number of state tables

Click on the arrows or type a number to represent the maximum number of state tables that the kernel is allowed to create before dropping any new connection request.

Note

When the number of state entries (i.e. the connections allowed by a stateful rule) reaches this number, no more stateful connections will be allowed, and you might experience that no new communications can be created until one or more NAT entries timeout and are deleted from the kernel list.

Loopback rules and stateful rules

Keeping the loopback rules (for packets generated and destinated to the local host) stateless, as in the default configuration of the policies (the one loaded when you first installed iqfire-wall) is strongly recommended, because in most cases you always want to allow the traffic between applications running on the local host.

Many graphical interfaces require the loopback communication and so keep the related rules off the state machine, so that in case it saturates as described above, the local communication between the applications running on your host are not affected.
