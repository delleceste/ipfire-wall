Rules

This window displays all the rules loaded into the firewall. They are organized in a tree with the aim of being grouped by owner, policy and direction, for faster browsing.
If you are not the administrator of the machine (root), you can browse the Admin tree, but in read-only mode. Actually, root can setup a group of rules which are always loaded into the firewall and cannot be deleted or modified by an unprivileged user.
The user tree instead can be browsed and modified.
Moving the mouse over each item in the tree, in the information panel above (if not active, click on the i icon in the tool bar) one can see the rule represented in a more readable html format, which summarizes the rule represented in the tree item.

Selecting a tree item will fix the information for that rule, and moving the mouse over other items will no more produce an update in the info panel.

Once an item is selected, right click on it if you want to:

  • add a new rule after the rule you're positioned on;
  • modify the selected rule;
  • delete the selected rule in the tree.
Remember to click on the Apply button to really update the rules in the firewall and then on the Save button, if you want to write the informations on the files.
If you are unhappy with your changes, just avoid applying or saving the rules and go back to another firewall window or click on the Undo button.

Inside the rule tree window, you can perform a search based on strings accepting wildcards (*), clicking on the menu View and then Search....
A suitable widget will appear on the top of the tree and will perform a search for you.
Just type the string you want to look for, e.g. 192.168.1.*, select in the combo box the field inside which you want to look for (in this case Source IP, and click on the button Find.
