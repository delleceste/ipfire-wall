Create a permission rule from the selected items

For each item selected, a new permission rule is added to the firewall.

The rule is built on the basis of the information represented inside the item:

  • direction (input or output);
  • protocol (TCP, UDP, ICMP, IGMP);
  • source and destination IP addresses;
  • source and destination ports;
  • network interface name;
  • the name of the rule you provide (or the default one, not advised!);
  • the possibility to be notified by a popup when a rule is applied by the firewall to a packet that corresponds to it.

Modifying the contents of a rule

Remember that you can modify the contents of the items in this window to create a more generic rule or a rule with different behaviour.

Keep also in mind that normally the source port is ignored when the firewall adds a rule from the entries listed in this popup. If you want the rule to check against the source port, manually provide it - if you know what this implies.

Color

The items selected and marked as Allow will be bold represented and will assume a dark green color.
