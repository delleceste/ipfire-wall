The output block

The packets in the output block will leave the machine through the next block: the post routing phase.

Here you can filter the network packets that are about to leave your computer, generated from a local application. For instance, you can allow the network traffic towards a specific service, as the web browsing or deny connections towards the peer to peer networks.

  • You can block a connection adding an output denial rule, or;
  • accept an outgoing connection adding an permission rule.
In the output direction, you can change the destination of a locally generated packet. For instance, you could redirect the connections towards the port 80, the web service, towards the proxy server listening on the port 8080. This can be done adding an output destination NAT rule
