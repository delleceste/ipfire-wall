Normal users can configure the firewall

A key feature of ipfire-wall/iqfire-wall is the possibility to allow any unprivileged user of the machine to configure the firewall.

Properties

  • The root's policies are loaded before the rules of the unprivileged user. This means that the normal users' rules are evaluated after the root's ones, and so have a lower priority;
  • a normal user cannot delete nor modify root's policies.

As a natural consequence, if the administrator permits a connection, a normal user can decide, for his own safety reason, to block it, but if root blocks a connection, a normal user cannot allow it because the firewall first scans and apply the explicit denial rules and then the permission ones.

Notwithstanding these important secure approach, the administrator is able to avoid a normal user personalizing the firewall with his own rules.

On the other hand, if the computer is for personal/familiar use, there ideally be just a few administrator's rules and many normal user defined policies.

An idea could be to adopt the default installation configuration, which contains a basic permission ruleset for an average user which navigates the web, reads the email, eventually connects to an ssh server, and to add by normal user account all the rest.

Another interesting scenario could be represented by a parent that, as root, configures a safe network environment and lets his children use and safely configure iqfire-wall, while learning linux internetworking.
