Log based on the connection state

This option provides yet less information to the console: not only identical packets aren't sent through the user/kernel communication channel, but only packets with different states are shown.

This means you will see for a connection only a packet notification per state: setup, setup ok, established, fin wait, time wait, close wait.. and so on.

Of course, the same information is presented again after the log timeout has elapsed. To tune such timeout, you must be root and change the settings in the Configuration page, section Advanced.
