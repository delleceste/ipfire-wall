/proc/sys/net/core/rmem_default

Should you encounter user/kernel communication problems, especially telling you recvfrom(): no buffer space available, you can try increasing this value, to enlarge the socket receive buffer size used to exchange a netlink packet between kernel and user space.

Do not change the default number unless you are very sure and you really encounter a problem like that described above. Inside iqfire-wall user interface, you should normally not run into this kind of trouble.
