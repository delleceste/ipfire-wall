Do not unload the rules at exit

Root might want to start iqfire-wall to edit the administrative ruleset and then close the root configuration session (i.e. close iqfirewall) and let normal users use the software, while leaving all the rules loaded into the kernel.
On the other hand, normal users' rules are always flushed from the firewall when the application is closed, and there is no way to change this behaviour.

On the other hand, if this feature is disabled, when the user interface is turned off all the rules are removed and this means:

  • no rules are loaded: the default policy (normally block) is applied;
  • every normal user can add his own rules (if allowed in the check box above) and they will be the only rules consulted.
