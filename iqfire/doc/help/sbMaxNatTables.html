Set the max table entries

For each Network address translation applied to a packet, the kernel uses a dynamic table to remember the data associated to the Nat operation on that packet.

Click on the arrows in the spinbox to change the value of the maximum number of NAT tables that you want to allow in the kernel internal tables.

Remember that, when the number of the NAT tables reaches the maximum allowed, NAT will stop working until one or more among the allocated entries expires and the memory associated is freed.

At the moment, the event mentioned above is logged through the system logger only.

Tune the number according to the amount of NAT traffic you expect in your network.
