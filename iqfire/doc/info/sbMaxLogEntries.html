Maximum number of log entries

As it happens with the NAT and state tables, when the log level between user and kernel is set to Smart Log: only unseen packets, for each information entry sent to the firewall console to notify a filtering or Nat action a table is created and memorized, so that the same information is not sent again to the user for a certain period of time.

The number of maximum log entries is tunable here. When it is reached, no more packets are notified to userspace and you might experience a silence period in the firewall console, until another entry decays and is removed from the internal list.
