Timeout of the log entries in kernel space

When a packet is printed on the firewall console and the Log level between user/kernel option is set to Smart Log: only unseen packets (on this window, in the section Verbosity level in kernel/user communication), an entry for that packet is allocated in the kernel space and, until that entry is alive in the firewall memory, the packet is not printed anymore on the console.

This feature avoids that identical information is repeated and sent between the user and kernel spaces, and reduces the load caused by the console printing system.

Tune this period to obtain a good compromise between performance and the need to view more or less often the same packet information.

The default setting is 20 seconds and this implies that, for instance, if the console prints a packet information like:

[OK 17]IN: [eth0] |TCP| 140.105.4.3:http-alt-->140.105.5.83:60322 |P|A|EST[me -> all the world!]

the same information will be repeated after this timeout has elapsed, if the connection above is still alive, obviously.
