Reset flag

The reset flag is used when a segment arrives that is not intended for the current connection. In other words, if you were to send a packet to a host in order to establish a connection, and there was no such service waiting to answer at the remote host, then the host would automatically reject your request and then send you a reply with the RST flag set. This indicates that the remote host has reset the connection.

Port scanning thanks to the reset flag

In most cases this feature is used by most hackers in order to scan hosts for open ports (i.e. services active on a machine). All modern port scanners (nmap for instance are able to detect open or listening ports thanks to the reset flag.
The method used to detect these active services bound to local ports is very simple: when attempting to scan a remote host, a valid TCP segment is constructed with the SYN flag set to true and sent to the target host. If there is no service listening for incoming connections on the specific port, then the remote host will reply with ACK and RST flags set to true, and all the others set to false. If, on the other hand, there is a service listening on the port, the remote host will construct a TCP segment with the ACK flag set. This is, of course, part of the standard 3-way handshake discussed in the syn section. Once the host scanning for open ports receives this segment, it will complete the 3-way handshake and then terminate it using the FIN flag, marking the specific port as "active".
