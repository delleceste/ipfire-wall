Maximum number of state tables

For each new connection tracked by the state machine, an associated state table is created inside the kernel.
This has been done to prevent denial of service attacks, that would create a huge amount of state tables entries that would saturate the system memory.

Keep in mind that, if you make large use of stateful rules, which actually is encouraged, the firewall will allow at most this number of connections at a time. If a new connection is requested and the number of state entries has reached the maximum allowed, the new connection will be dropped.
So it is important to tune this number to the amount of traffic expected in your network keeping it a bit higher than the maximum peak possible in your computer network. A first period of observation could be a good choice to finally tune this number to the optimum.
