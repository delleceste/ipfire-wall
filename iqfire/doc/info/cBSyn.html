The TCP SYNchronization flag

The tcp SYN flag is used to initialize a connection.
If it is the only flag set to true, the packet represents the first packet in a connection, i.e. a connection request.
If instead it is set to true together with the ack flag, it represents a connection accepted packet, in response to a previous connection request.

For example, if you require a web page with your browser, an outgoing connection request is sent and an incoming connection accepted response packet is expected.

Observation

Enabling the syn check here and choosing On in the radio button nearby is a good idea to allow a well initiated tcp connection. Actually, all TCP connections should start with the syn flag on and all other flags off. Imposing this on your rule should guarantee that the connection is initiated cleanly.
To check against all this flags, you should enable all the other check boxes and select off for all the other flags except syn.
Moreover, this advanced packet filtering based on the verification of the syn tcp flag requires a stateful rule to work correctly after the connection has been setup.
Actually, if the first packet is accepted thanks to the syn flag set to true, it is necessary to associate a state table to the new connection to guarantee the data transmission in both directions.
