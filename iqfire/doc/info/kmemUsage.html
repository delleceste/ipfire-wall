Kernel memory usage for internal tables

This plot displays the kernel memory usage over time. Five curves are drawn on the graph:

  • state tables curve: depicts the memory usage, in kilobytes of the state tables loaded into the kernel by the firewall. If allowed traffic is mainly managed by stateful rules, then this curves indicates the trend of the allowed traffic against time;
  • source nat tables curve: depicts the memory usage, in kilobytes of the source nat tables loaded into the kernel by the firewall. Also masqueraded packets pertain to this trend;
  • destination nat tables curve: depicts the memory usage, in kilobytes of the destination nat tables loaded into the kernel by the firewall;
  • log info tables curve: depicts the memory usage, in kilobytes of the log info tables loaded into the kernel by the firewall. High values for this quantity reveal high activity of the firewall console.
    To reduce this value, enable silent modality, if you are not interested in reading the firewall activity through the console, or decrease the log entries lifetime in the advanced settings of the firewall (root only);
  • total curve: represents the sum of the values of the other curves and reveals the total memory in use for the firewall dynamic tables.

These curves, especially the state tables one, reveal the firewall activity and the number of connections managed by the firewall, particularly the allowed traffic (state tables). The quantity of memory used by the kernel for these tables varies in time in consequence of the network activity.
Note: file sharing applications open a great number of connections and might produce elevated values of the curves.

Displaying details on curve values

Click with the left button of the mouse on a point of the plot near a curve to display a marker with the details of the closest point of the closest curve.

Zoom is available on the plot too. Select a region with the mouse left button pressed and then release the mouse button to plot the selected area.
