Leave rules loaded at exit

If you enable this feature, then when ipfire-wall or iqfire-wall exits, it leaves all the rules loaded into the kernel, including the kernel module.

Note that this option is fundamental to let the other users configure the firewall by themselves, while respecting the administrator's decisions.

Note

This option affects also the command line interface ipfire-wall.
If you leave the rules loaded, then each unprivileged user's rule will be added to the end of the root ruleset, thus granting the administrator's policies to have the first priority.

On the other hand, if this feature is disabled, when the user interface is turned off all the rules are removed and this means:

  • no rules are loaded: the default policy (normally block) is applied;
  • every normal user can add his own rules (if allowed in the check box above) and they will be the only rules consulted.

Important note

It is important that the administrator configures the firewall as root only adding the policies strictly affecting the system services that require a network connection and the other fundamental services which in every circumstance he wants to be applied, let's say from system startup to system shutdown.

For every other kind of service, especially those specific to the normal users of the computer, let each unprivileged user configure his own policies.
Moreover, normal user's policies are always removed from the firewall when the user interface (be it ipfire-wall or iqfire-wall) is turned off.
