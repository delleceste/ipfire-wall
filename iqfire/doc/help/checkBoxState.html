Stateful rules

Enable this to associate a state table to the rule and turn on the simple state machine provided by iqfirewall. Read the info panel above for further information or the iqfirewall manual. If you disable this feature, remember that the rule you are creating or modifying will allow the flow in only one direction (i.e. input but not output, or viceversa), and so you will probably need another rule to allow the same flow in the opposite direction.

    For instance, if you are allowing the output traffic towards 195.110.58.52, port 22:
  • with a stateful rule, this rule is enough to assure that the connection will be accepted and the data exchanged in both directions;
  • without the state machine, you will have to add another rule allowing the incoming traffic from source 195.110.58.52 and source port 22
    • .
In the second case, if the rule is not well configured, you will not be protected against new connections initiated by 195.110.58.52, source port 22.

Note

The state machine must be globally enabled by the administrator in the firewall settings: Settings->Configure the firewall interface->Advanced tab
